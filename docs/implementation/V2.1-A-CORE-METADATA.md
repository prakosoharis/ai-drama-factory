# V2.1-A Core Metadata and Validation

V2.1-A is a local, read-only foundation for discovering a creative project,
loading its JSON metadata, validating relationships, and displaying project or
shot information. It is file-based and uses Python 3.9+ standard library only.

## Local installation

No sudo, GPU, network access, or third-party dependency is required.

```sh
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --editable .
```

## Commands

```sh
drama inspect --project examples/v2.1-minimal-project
drama validate --project examples/v2.1-minimal-project --level structure
drama validate --project examples/v2.1-minimal-project --level files --format json
drama shot show shot-001 --project examples/v2.1-minimal-project
```

`--project` accepts a project directory or `drama-factory.project.json`. If it
is omitted, discovery walks upward from the current directory only. Exit codes
are `0` valid, `1` validation errors, `2` invalid command/configuration, and
`3` unexpected failure. Add `--debug` for a traceback on unexpected failures.

`STRUCTURE` validates metadata, IDs, timestamps, paths, enums, and references.
`FILES` additionally requires non-empty candidate media and verifies supplied
SHA-256 checksums. The synthetic example intentionally passes structure and
fails files because it includes no media.

## Example project

`examples/v2.1-minimal-project/` contains synthetic Shot 001 metadata: two
plans and jobs, a rejected first candidate with a timecoded head-motion review,
an approved second candidate, rough-cut selection, and Cut Manifest. It has no
V1 assets and no production media. It is a neutral demo project, not an import
or dependency on `project-film`.

## Intentionally deferred

No rendering, GPU execution, media decoding, review/selection mutation,
assembly, migration, database, web UI, distributed queue, or automatic repair
is implemented. FILES validation verifies file presence, non-emptiness, and
SHA-256 only; it does not decode video yet.
