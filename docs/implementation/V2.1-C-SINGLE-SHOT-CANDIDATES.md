# V2.1-C Single-Shot Candidates

V2.1-C creates immutable Render Plans, local mock Render Jobs, and
`REVIEW_REQUIRED` Candidates for one shot. Use `drama render plan create`,
`drama render run`, `drama candidates`, and `drama candidate show`.

The mock adapter writes `candidate.mock`, never claims video output, and stages
under `.drama-factory/temp` before atomically publishing a candidate directory.
Per-shot atomic lock files prevent concurrent allocation. Failed jobs are
marked `FAILED` and never publish candidates. Candidate metadata includes
checksums, QC, and provenance. Wan/GPU execution, approval, selection, and
assembly remain deferred.
