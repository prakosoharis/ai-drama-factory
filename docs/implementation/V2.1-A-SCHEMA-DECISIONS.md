# V2.1-A Schema Decisions

## Version and format

V2.1-A supports JSON objects with `schema_version: "0.1"`. This is an
implementation-ready internal version, not a permanent public standard.
Unknown fields remain in loaded metadata and are not discarded.

## Fields and IDs

All entities use the field names in the approved V2.1 design. Required fields
are validated as present; nullable values are permitted only where a value is
not yet known (for example cost, a missing legacy prompt artifact, or optional
audio component). Entity metadata declares `entity_type` for directory scanning.

Human-readable IDs are validated as follows: `nightmare-film`, `scene-001`,
`shot-011`, `rp-shot-011-001`, `job-shot-011-001`, `cand-shot-011-001`,
`rev-cand-shot-011-001-001`, `sel-shot-011-001`, `cut-rough-002`, and
`pkg-shot-011-001`.

## Timestamps, paths, and enums

Timestamps must be timezone-aware ISO 8601 UTC; canonical output is
`2026-07-15T18:30:00Z`. Paths are relative to the creative project root, may
not contain variables, absolute paths, `..`, or hidden runtime areas, and are
resolved to detect escape where practical. FILES validation checks candidate
media existence/non-emptiness and a supplied SHA-256 checksum.

Enums are closed for V2.1-A render-job, candidate, review, selection, and cut
states. Candidate status cannot contain `SELECTED_FOR_CUT`; selection is a
separate entity. `model_specific` remains a namespaced object (for example
`{"wan": {...}}`).

## Immutability and limitations

The validator does not mutate metadata. Candidate, Render Plan, Shot Package,
and Cut Manifest immutability is a workflow rule that later write commands
must enforce. V2.1-A does not decode media, validate scene registries, infer
review decisions, require every optional binary, or implement migration.

## Migration expectation

Legacy V1 assets must be registered later through new sidecar metadata in a
separate workspace, with V1 source paths/checksums and unknown provenance
explicitly recorded. V2.1-A never reads or changes `project-film`.
