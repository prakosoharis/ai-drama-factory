# V2.1-G Optional Migration Plan: project-film V1

## Status

Deferred until V2.1-G. Native V2 work and tests must not depend on this plan.

## Objective

Adopt V2.1 metadata gradually while preserving `project-film` at
`v1.0-baseline`. Migration must not move, rename, modify, or regenerate V1
video, audio, raw renders, final films, model weights, or logs.

## Recommended approach

Use a separate V2-compatible project workspace with a Project Manifest and
metadata sidecars that reference V1 assets by read-only relative import mapping
plus recorded checksums. Do not write sidecars into the frozen V1 repository.
The migration importer is a future command experience, not a V2.1
implementation requirement.

The migration source is the immutable V1 tag. Its provenance records the
repository identifier, `v1.0-baseline`, source relative path, detected checksum,
import timestamp, and any unknown field. An absolute local path may be used by
a non-canonical local import configuration, but never stored as canonical
manifest data.

## Mapping

| V1 source | V2.1 representation | Migration constraint |
| --- | --- | --- |
| Shot list, story, and existing shot metadata | Draft Shot Contract, clearly marked as imported and requiring human confirmation | Do not infer missing dramatic intent silently. |
| Existing shot images | Reference artifact with source path and checksum | Never duplicate unless an approved archival copy policy exists. |
| Dialogue and other audio | Audio dependency/reference or later Shot Package component | Preserve source identity and synchronization limitations. |
| Wan prompts | Derived imported prompt artifact and provenance only | Prompts do not become canonical Shot Contracts. |
| Raw videos | Baseline Candidate with `render_plan_id: UNKNOWN` where reconstruction is impossible | Checksum and decode before registration; no rerender needed. |
| Existing final assembly | Historical delivery reference, not a V2 Cut Manifest by assumption | Reconstruct a cut only after timeline verification. |
| Logs | Provenance evidence attached to import or job history | Mark unavailable fields as `UNKNOWN`, never fabricate cost/model details. |

## Phased path

1. Create an independent V2-compatible workspace and Project Manifest that
   identifies `project-film@v1.0-baseline` as an external historical source.
2. Inventory V1 artifacts read-only and calculate checksums through a future
   validation/import operation.
3. Register raw Shot 11 video as a baseline Candidate only after media and
   checksum validation. Mark incomplete provenance explicitly.
4. Create or human-confirm the Shot 11 Contract from V1 creative sources.
5. Create the first V2.1 Render Plan for a new Shot 11 candidate. It is a new
   attempt and does not alter baseline Candidate metadata.
6. Migrate other shots only when a revision, comparison, or assembly need
   justifies the metadata work.
7. Construct a V2 Cut Manifest only from verified per-shot candidates and
   timeline data. Until then, treat the V1 final film as historical delivery.

## Import rules and exceptions

- Imports are append-only: an import creates new metadata records, never edits
  the V1 source or an earlier import record.
- Asset registration validates that the source exists, is readable, decodes
  where appropriate, and matches its recorded checksum.
- A baseline Candidate may be `REVIEW_REQUIRED` even when the V1 film used it;
  historical inclusion is not current picture approval.
- Unknown model version, seed, cost, duration, or job timing is recorded as
  `UNKNOWN` with a reason. It is not guessed from filenames.
- If the V1 source becomes unavailable or changes checksum, registration and
  selection are blocked until the exact historical source is restored.

## Deferred implementation decision

The future import interface may be a manifest-driven import operation with
sidecar generation. It should not be a mandatory bulk migration command: a
bulk import would imply unnecessary production scope and risk. The first
validated migration slice is Shot 11 plus only the dependencies required for a
rough-cut replacement.
