# V2.1 Open Questions and Approval Gates

## Open questions

1. What exact authority model resolves conflicting human reviews, and may a
   Review Critic only comment or also request revision?
2. Which Shot Contract statuses and approval gates exist before a Render Plan
   may be created? This design intentionally does not finalize them.
3. Is a rejected candidate eligible for `BENCHMARK` or `PREVIEW` selection,
   and what visual labeling must prevent accidental use in a final cut?
4. What media checksum algorithm and canonicalization policy should be used?
5. Are proxies stored beside candidates or generated on demand after future
   storage constraints are understood?
6. What constitutes a complete audio package for V2.1, especially when V1
   dialogue is embedded in an S2V output?
7. Which Cut Manifest timebase, timecode convention, and transition vocabulary
   should be canonical?
8. What retention policy applies to temporary outputs, logs, proxies, and
   failed render evidence?
9. What compatibility and migration policy applies when Project Manifest or
   entity schema versions evolve?

## Risks

- Treating imported V1 data as complete may create false provenance.
- A permissive selection policy could place rejected or checksum-invalid media
  into an editorial cut.
- File-based concurrency without disciplined locks can produce ID collisions or
  partial records.
- Checksumming large media can be operationally expensive; skipping it weakens
  reproducibility.
- V2.1 can become over-designed if it attempts continuity, multi-renderer, or
  distributed-execution concerns before the Shot 11 path is proven.

## Decisions requiring human approval

- The canonical metadata format and schema-validation library, before any
  implementation begins.
- Contract approval roles and review conflict-resolution authority.
- Whether baseline V1 candidates may receive special historical selection
  purposes, and their required labels.
- Checksum algorithm, retention/storage policy, and canonical timebase.
- The first V2-compatible project workspace location and read-only method for
  accessing the V1 baseline.
- Whether a first implementation includes proxy generation or requires proxies
  to be supplied externally.

## Explicit non-decisions

This design does not select a database, renderer SDK, worker runtime, queue,
web UI, model routing policy, authentication mechanism, or final schema.
